## 提示词

```
基于微信读书数据，分析用户的读书特点，识别信息茧房和认知局限，并推荐扩展认知的书单。

## 执行步骤

### 第 1 步：获取阅读统计数据

调用 `/readdata/detail` 接口，获取以下数据：

- 总阅读时长（totalReadTime）
- 阅读天数（readDays）
- 读过本数、读完本数（readStat）
- 笔记数量
- 偏好分类（preferCategory）— 按阅读时长排序
- 偏好作者（preferAuthor）
- 读得最多的书（readLongest）
- 偏好时段（preferTime）
- 文字阅读占比（readRate）

### 第 2 步：获取书架信息

调用 `/shelf/sync` 接口，获取：

- 书架总量（books.length + albums.length）
- 分类分布（按 category 统计）
- 最近阅读的书（按 readUpdateTime 排序）
- 是否读完（finishReading）

### 第 3 步：分析读书特点

从以下维度分析：

1. **阅读习惯**：稳定性、笔记量、完成率
2. **阅读偏好**：按阅读时长排序的分类占比
3. **阅读品味**：读得最久的书、偏好作者
4. **书架构成**：各类别占比可视化

### 第 4 步：识别信息茧房

检查以下 5 个常见盲区：

1. **叙事思维 vs 科学思维**：是否全是故事类书籍，缺少科学方法论
2. **本土视角 vs 全球视野**：是否只有中国内容，缺少世界史和全球视角
3. **感性 vs 理性**：是否缺少经济学、政治学、社会学、心理学的严肃著作
4. **成长深度**：个人成长类是否停留在鸡汤层，缺少批判性思维和元认知
5. **舒适区**：是否缺少让人「不舒服」的书（挑战世界观的书）

### 第 5 步：推荐书单

针对每个识别出的盲区，推荐 5 本书：

- **建立科学思维**：进化论、认知科学、数据思维、概率思维
- **拓展全球视野**：世界史、全球化、非西方文明
- **理解社会运行**：经济学、政治学、社会学
- **升级思维系统**：批判性思维、系统思维、元认知
- **打破舒适区**：反乌托邦、群体心理、媒体批判

每本书说明「为什么读」——与用户的认知盲区的关联。

### 第 6 步：输出阅读策略

给出具体的行动建议：

- 调整阅读比例（当前 vs 建议）
- 提高完成率的方法
- 建立季度主题阅读计划
- 如何保持文学阅读的同时拓展边界

### 第 7 步：输出阅读画像

用可视化方式展示用户的阅读特征占比。

## 输出格式

按以下结构输出 Markdown 文档：

1. 阅读数据概览（表格）
2. 读书特点分析（习惯、偏好、品味、书架构成）
3. 信息茧房与认知局限（5 个盲区，每个包含现状、缺失、风险）
4. 推荐书单（5 组，每组 5-6 本，含书名、作者、为什么读）
5. 阅读策略建议（比例调整、完成率、主题阅读）
6. 阅读画像（可视化占比）
7. 总结（一句话核心建议）

## 注意事项

- 阅读时长字段单位是秒，需转换为「X小时Y分钟」
- 书架数量必须包含 albums（有声书）
- 偏好分类按 readingTime 降序排列
- 推荐书单要考虑用户已有的书（不重复推荐）
- 分析要客观，茧房识别要有具体依据，不能泛泛而谈
```

---

## 使用方式

### 方式一：直接使用

复制上面的提示词，在对话中粘贴使用。

### 方式二：保存为 Skill

将提示词保存为 `~/.claude/skills/reading-analysis/SKILL.md`，然后通过 `/reading-analysis` 调用。

### 方式三：保存为 Command

将提示词保存为 `~/.claude/commands/reading-analysis.md`，然后通过 `/reading-analysis` 调用。
